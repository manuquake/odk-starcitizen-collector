# Privacy

ODK Star Citizen Collector is a local companion collector for Star Citizen.

## What It Reads

- The configured Star Citizen `Game.log` file.
- The `ScreenShots` folder near the configured game installation, if screenshot
  proof uploads are enabled.
- Its own local configuration file:

```text
%APPDATA%\odk-control-center\starcitizen-collector.json
```

## What It Sends

Depending on configuration and backend support, the collector can send:

- parsed game-log events;
- heartbeat and collector connection state;
- parser and collector version;
- machine label configured by the user;
- screenshot proof files and metadata when screenshot proofs are enabled.

Screenshot metadata can include file name, relative path, size, SHA-256 hash,
and observed timestamp.

## What It Does Not Do

- It does not read Star Citizen process memory.
- It does not inject code into Star Citizen.
- It does not modify Star Citizen files.
- It does not read Discord browser cookies or passwords.
- It does not contain a preinstalled collector token.

## Local Token Storage

The collector token is generated by the backend and saved locally in:

```text
%APPDATA%\odk-control-center\starcitizen-collector.json
```

Do not publish this config file. If a token is exposed, revoke it from the ODK
backend and generate a new one.

## Screenshot Proofs

Screenshot upload is intended for operational proof galleries. If enabled, the
collector can upload screenshots from the configured Star Citizen `ScreenShots`
folder.

Disable screenshot proofs in config if you do not want screenshot uploads:

```json
{
  "screenshotProofsEnabled": false
}
```
