# ODK Star Citizen Collector

Local Windows-friendly collector for Star Citizen `Game.log`.

The collector watches the local game log and sends near real-time events to an
ODK backend that exposes the `Star Citizen / Entity Live` page.

This project is intentionally simple and transparent:

- it reads `Game.log`;
- it can watch the `ScreenShots` folder for proof uploads when enabled;
- it sends collector heartbeat and parsed events to the configured backend;
- it does not read Star Citizen process memory;
- it does not inject code into the game;
- it does not modify Star Citizen files.

## Status

Current package version: `0.2.0`

Current parser version: `mvp-0.3.0`

This is an MVP companion collector. The parser is intentionally conservative and
heuristic because Star Citizen does not expose a stable public live-character
state API.

## Requirements

- Windows 10/11
- Node.js 20+ for source usage
- A collector token generated by an ODK backend
- A local Star Citizen installation with `Game.log`

## Quick Start From Source

Install Node.js, then run:

```powershell
npm run collector:init -- --server-url https://app.odkclan.it
```

The command asks for:

- `Collector token`
- `Client ID`
- machine label
- `Game.log` path

Then start the collector:

```powershell
npm run collector:run
```

The local config is saved in:

```text
%APPDATA%\odk-control-center\starcitizen-collector.json
```

## Build Windows Installer

The Windows installer is built from this repository using Inno Setup 6.

Requirements:

- Node.js available as `node.exe`;
- Inno Setup 6 compiler `ISCC.exe`;
- PowerShell.

Build locally:

```powershell
npm run build:windows-installer
```

Output:

```text
release\ODK-StarCitizen-Collector-Setup.exe
release\ODK-StarCitizen-Collector-Portable.zip
release\SHA256SUMS.txt
```

## Server URL

Both forms are accepted:

```text
https://app.odkclan.it
https://app.odkclan.it/api/starcitizen/collector
```

Do not use Discord OAuth URLs, bot invite URLs, or URLs with query strings.

## Event Types

The collector can derive events such as:

- `session_started`
- `session_ended`
- `server_joined`
- `location_changed`
- `ship_changed`
- `quantum_started`
- `quantum_ended`
- `quantum_target_selected`
- `quantum_route_calculated`
- `quantum_error`
- `mission_accepted`
- `mission_objective_updated`
- `mission_completed`
- `mission_failed`
- `mission_abandoned`
- `hauling_objective_created`
- `loadout_item_attached`
- `insurance_claim_requested`
- `insurance_claim_completed`
- `player_spawned`
- `client_error`
- `screenshot_captured`

## Known Limits

`Game.log` does not reliably expose:

- mission payout in aUEC;
- net session profit;
- exact cargo quantities sold;
- final inventory or cargo content.

Those fields should be handled with OCR, screenshots, or quick manual
confirmation in a companion UI if needed.
